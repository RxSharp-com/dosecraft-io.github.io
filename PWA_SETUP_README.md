### Step 2 — Patch game.html

Open `game.html`. Inside `<head>`, paste the tags from **PATCH_game.html.txt**,
starting at the `<link rel="manifest">` line, down to and including the `</style>` tag.

Then, just before `</body>`, paste the `<script>` block from the same file
(the section marked "PASTE BELOW BLOCK JUST BEFORE </body>").

**If you already have a `<meta name="viewport">` tag**, do not add a second one.
Instead, edit the existing tag to include these attributes:
```
maximum-scale=1.0, user-scalable=no, viewport-fit=cover
```

### Step 4 — Commit and deploy

```bash
git add manifest.json sw.js icon-192x192.png icon-512x512.png game.html index.html
git commit -m "feat: add PWA support (manifest, service worker, icons)"
git push
```

GitHub Pages and Netlify will serve `sw.js` automatically. No build step needed —
the SW is a plain JavaScript file.

### Companion Mode timer — why it's safe

The infusion session timer is stored in:
1. `localStorage` (`sessionStartTime`, `sessionActive`)
2. React component state

The service worker **only caches static file bytes**. It never reads or writes
`localStorage`. Installing or updating the SW cannot cause the timer to reset.
The `endInfusionSession()` function remains the only code path that clears the timer.

---

### Service worker cache invalidation

When you deploy a new version of `InfusionArcade.js`, users with the old SW
will get stale JS until the SW updates. To force an immediate update:

1. Open `sw.js`
2. Change `const CACHE_VERSION = 'v1.0.0'` to `'v1.0.1'` (or any new string)
3. Commit and push

The browser will detect the changed SW file, install the new version, delete
the old cache, and serve fresh files — all automatically.

### GitHub Pages path note

`start_url` in `manifest.json` is set to `/game.html`.

If your GitHub Pages site is served from a **project repo subpath**
(e.g., `rxsharp22.github.io/dosecraft/`) rather than a custom domain
(`dosecraft.io`), you need to update `start_url` and the SW scope:

```json
// manifest.json — subpath version
"start_url": "/dosecraft/game.html",
"scope": "/dosecraft/"
```

```js
// index.html / game.html — SW registration for subpath
navigator.serviceWorker.register('/dosecraft/sw.js', { scope: '/dosecraft/' })
```

If you're using the custom domain `dosecraft.io`, the `/` paths in the
current files are correct and no changes are needed.

### How to verify the PWA is working

1. Deploy to GitHub Pages / Netlify
2. Open Chrome DevTools → Application → Manifest
   - Should show name, icons, start_url
3. Application → Service Workers
   - Should show `sw.js` as "activated and running"
4. Application → Cache Storage
   - Should show `infusion-arcade-v1.0.0` with your core files listed
5. On iPhone Safari: tap Share → Add to Home Screen
   - Icon should appear; tapping it should open in standalone mode (no Safari chrome)

### iOS-specific behavior notes

- `apple-mobile-web-app-capable: yes` enables standalone mode when launched from Home Screen
- `apple-mobile-web-app-status-bar-style: black-translucent` extends the game
  content behind the status bar for a full-bleed look
- `viewport-fit=cover` + `overflow: hidden` on `body` prevents bounce-scroll
  and pull-to-refresh from interrupting gameplay
- The canvas `touch-action: none` patch prevents iOS from interpreting
  gameplay swipes as page-scroll gestures
